# Datasheet for PI-JEPA Synthetic 3D Darcy + Multi-Fidelity Tier 1

Following Gebru et al. ([arXiv:1803.09010](https://arxiv.org/abs/1803.09010)).
This datasheet covers the synthetic data generated for the PI-JEPA follow-up
paper. The third-party datasets we use (CCSNet, FNO4CO2, Fourier-MIONet,
PDEBench ADR, SPE10) ship their own datasheets / paper-style documentation;
this datasheet does NOT replace those.

---

## Motivation

**For what purpose was the dataset created?**
To enable controlled, reproducible smoke testing and ablations of the
PI-JEPA architecture on a 3D coupled-physics problem where the solver
behavior is well-understood. Specifically:
- Tier-2 (fine, expensive) synthetic Darcy: a controlled stand-in for
  real 3D reservoir simulation outputs while we wait on OPM Flow / SPE10.
- Tier-1 (coarse, cheap) synthetic Darcy: the multi-fidelity intermediate
  tier that exercises the paper's multi-fidelity contribution.

**Who created the dataset?** This repository's maintainer (PeterLi-jpg),
as part of the PI-JEPA follow-up paper.

**Who funded the creation?** Self-funded research; no external grant.

---

## Composition

**What do the instances represent?**
Each instance is one realization of the steady-state single-phase
Darcy problem on the unit cube:
```
-∇·(K(x) ∇p(x)) = q(x)     in Ω = [0,1]^3
                p = 0       on ∂Ω  (Dirichlet)
```
where K is a log-Gaussian random field (LGRF) sampled spectrally and
q is a centered Gaussian "injection" source.

**Schema:**
- Tier-2 (`data/darcy_3d/darcy3d_{train,test}.pt`):
  - `x`: torch.float32, shape (N, 1, 32, 32, 32) — permeability K (after
    log-transform from LGRF, exponentiated to ensure positivity, then
    per-channel normalized).
  - `y`: torch.float32, shape (N, 1, 32, 32, 32) — pressure p.

- Tier-1 (`data/darcy_3d_tier1/darcy3d_tier1.pt`):
  - `x_coarse_native`: shape (N, 1, 16, 16, 16) — coarse permeability (raw)
  - `y_coarse_native`: shape (N, 1, 16, 16, 16) — coarse pressure (raw)
  - `x_upsampled`: shape (N, 1, 32, 32, 32) — coarse K trilinearly upsampled
  - `y_upsampled`: shape (N, 1, 32, 32, 32) — coarse p trilinearly upsampled
  - `metadata`: dict with `tier`, `coarse_resolution`, `target_resolution`,
    `n_samples`, `cg_rtol`, `cg_maxiter`, `length_scale`, `variance`, `seed`.

**Size?** Tier-2 default: 64 train + 8 test (18 MB total). Tier-1 default:
64 samples (18 MB). Both regenerable in minutes via the included scripts.

**Are there labels?** Yes — `y` is the deterministic solver output, treated
as a regression target during fine-tuning.

**Is there a recommended train/test split?** Yes — separate `*_train.pt`
and `*_test.pt` files generated with different RNG seeds (default 42 and
43 respectively). No validation split is bundled; downstream code carves
one out of train as needed.

**Are there errors / noise / redundancies?** The solver is a
preconditioned-CG on a 7-point Poisson stencil (`scipy.sparse.linalg.cg`).
Tier-2 uses tight tolerance (`rtol=1e-6`); Tier-1 uses loose tolerance
(`rtol=1e-3`, `maxiter=500`) to *intentionally* introduce solver error —
this is the whole point of the multi-fidelity contribution. We do not
filter samples by solver convergence (so a small fraction may not have
converged); this matches industry practice for cheap-tier reservoir runs.

**Are there sensitive attributes?** No — fully synthetic.

## Collection process

**How was the data acquired?** Generated by `scripts/generate_darcy_data_3d.py`
(Tier-2) and `scripts/generate_darcy_tier1.py` (Tier-1). Both are pure-Python,
deterministic given the RNG seed, and re-runnable on any machine with the
included `setup.sh`.

**Who collected it?** Code in this repository; no human in the loop.

**When?** Generated at run time; metadata fields capture the parameters
used. The committed `.pt` files in `data/darcy_3d/` and
`data/darcy_3d_tier1/` were last generated on 2026-05-23 with the seeds
documented in the metadata.

---

## Preprocessing / cleaning / labeling

**Was any preprocessing done?**
- Tier-2: per-batch mean/std normalization on `x` and `y` separately
  (mean and std saved with the dataset).
- Tier-1: same plus trilinear upsampling from 16³ to 32³ to match the
  encoder's expected resolution.

**Was the raw data saved?** Tier-1 keeps both the raw coarse-native
arrays AND the upsampled arrays in the same .pt file. Tier-2 only saves
the post-normalization arrays (regenerating from RNG seed reproduces
the raw arrays exactly).

## Uses

**For what tasks could the dataset be used?**
- 3D Fourier-encoder smoke/ablation testing
- Operator-split JEPA pretraining (Tier 0 inputs only)
- Multi-fidelity pretraining (Tier 0 + Tier 1)
- Supervised baseline training (FNO3D, U-FNO3D, PINO3D — all included)
- Sample-efficiency curves (fine-tune sweep)

**For what should the dataset NOT be used?**
- Drawing conclusions about real CO₂ storage performance (use CCSNet /
  FNO4CO2 / Fourier-MIONet / SPE10 for that)
- Benchmarking against state-of-the-art solver runtimes (the Python CG
  solver here is intentionally simple, not optimized)

## Honest limitations

- Single-phase Darcy is **simpler** than the actual CCS problem (which
  is multiphase compositional). This dataset exercises the geometry and
  operator-splitting code paths cleanly but does NOT exercise capillary
  effects, phase changes, or chemistry.
- 32³ is small — the encoder is not stress-tested at production scale.
- Periodic-BC assumptions in spectral physics residuals approximate the
  Dirichlet BC of the solver; mismatch contributes a small bias that
  vanishes when we move to truly periodic problems (or use the FD
  residual variant).
